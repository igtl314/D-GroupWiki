=====Klägget=====

Kläggsamlingen är en samling av kontaktinformation till alla festerier som D-Group brukar göra. Kläggsamlingen har förr i tiden huvudsakligen getts ut i en tryckt upplaga gjord för att passa i en filofax, men de senaste åren har vi endast haft den digitalt. 

Mailet angående insamling av information bör helst skickas ut redan innan sommaren, allra senast i början av sommaren. Det brukar vara chefen som skickar ut mailet till de andra festerierna, men det skadar inte att påminna denna om det. Se till att ha en utsatt deadline, om du inte säger när informationen senast ska vara inlämnad kommer det ta låång tid. Många svar kommer ofta precis innan deadline, så man behöver inte bli helt förstörd om man inte fått mer än ett fåtal svar någon vecka innan. Ett påminnelsemail skadar dock inte. En sak vi missade var att man borde skicka med en mall som skall fyllas i så att du får sakerna i rätt ordning. Namn, telnr, mail, post. Det vore idealt om denna var klar innan nolle-p. 

Du tömmer databasen på all gammal info (har du jättetur så kommer festerierna själva uppdatera sin info om de fortf. kan sitt lösenord men räkna inte med det, förra året gjorde bara FM det) och sedan lägger du in festerierna allt eftersom du får deras info. Om du väntar med att lägga in alla till samma tillfälle kommer du vilja döda dig själv, just sayin'.

När du tycker att det är tillräckligt många som finns med i klägget för att folk ska ha nytta av den kan du börja skicka ut inloggningsuppgifter till alla festerier. Ge dem olika lösenord, festeristerna kommer busa med varandra annars.

I klägget finns också en kalender där festerifesterna ska finnas. Jag mailade bokningsansvarige i FörenadeFesterier och fick hans kalender och bara la in den i admin.d-group.se kalendern. Denna kalendern ska hållas hemlig och får endast ses av festerister/FF. (Alla festeristfester är mycket svarta och man vill inte att allmänheten vet om dem).

=====MySQL=====

För att logga in : mysql -u root -p052c8f85fd7d6cf316b5315235544747

För att köra ett kommando avslutar man med semikolon.

Kommandon :

SHOW DATABASES - Visar alla databaser.

USE d_group - Går in i d_group databasen. Man använder också USE för att byta databas.

SHOW TABLES - Visar tabeller i d_group databasen.

SHOW COLUMNS FROM users - Visar de olika egenskaperna för users.

DESCRIBE users - Visar också egenskaperna för users.

SELECT name FROM users WHERE festeri=2 - Visar d-groups användare.

USE d_group -> SELECT * FROM festerier - Hela klägget.

USE d_group -> SELECT name, level FROM users WHERE festeri=2 - Visar alla groupies och deras level.

INSERT INTO "d_group1314" SELECT * FROM user WHERE id BETWEEN 238 and 251 : Slänger in de användare som har id 238-251 i databasen d_group1314.

CREATE TABLE d_group1314 LIKE users : För att skapa en ny tabell med samma kolumner som users.

Groupies små porträtt på hemsidan som sorteras i ordning beroende på deras id. Därför kan du vilja ändra deras id för att på så sätt ändra deras position på hemsidan. Det gör du på följande sätt:

UPDATE users
-> SET id=289
-> WHERE name='Harald Stampe Pettersson';

_VÄLDIGT_ viktigt att du har med ett 'WHERE' villkor då du annars kommer ändra id't på allt som finns i hela tablet users vilket skulle bli väldigt kaos.

såhär: update users set festeri=2 where username='groupie-name';
